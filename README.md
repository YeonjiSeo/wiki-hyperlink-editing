# Editorial Trajectories in Wikipedia Reflect Underlying Hyperlink Structure

This repository contains the supplementary code and data processing pipeline for the research paper:  
**"Editorial Trajectories in Wikipedia Reflect Underlying Hyperlink Structure"**

## 📌 Terminology Note
* **Article vs. Page**: In the accompanying paper, we use the term **"article"** to refer to Wikipedia entries. However, in the source code and datasets, the term **"page"** is used to maintain consistency with the Wikipedia database schema and technical implementation. These two terms refer to the same entity.

## 🚀 Analysis Workflow
The analysis is structured into three sequential Jupyter Notebooks. They must be executed in order to maintain data dependency.

### 1. [Step 1] Preprocessing
* **File**: `[YJ_Wiki]Step1_Preprocessing.ipynb`
* **Description**: Extracts and cleans raw Wikipedia XML/SQL dumps. It filters data based on activity thresholds (e.g., minimum 1,000 edits per page and editor) and generates the "conditioned history" and "conditioned pairs" datasets that serve as the foundation for the entire project.

### 2. [Step 2] Inter-event Time (IET) Analysis
* **File**: `[YJ_Wiki]Step2_Inter_event_time.ipynb`
* **Description**: Implements the temporal analysis of editorial dynamics.
    * **IET Calculation**: Computes Overall, Immediate, and Long-term IETs for both hyperlinked and non-hyperlinked control groups.
    * **Complexity Analysis**: Provides a quantitative justification for the 1:1 sampling strategy by comparing its efficiency against exhaustive enumeration.

### 3. [Step 3] Hyperlink Network & Editor Classification
* **File**: `[YJ_Wiki]Step3_Hyperlink_network_and_editors.ipynb`
* **Description**: Integrates structural data with behavioral classification.
    * **Community Detection**: Coarse-grains the hyperlink network into topical communities using the Leiden algorithm.
    * **Diversity Measures**: Calculates Shannon Entropy ($H$) and the Inverse Simpson Index ($1/\lambda$) for editors.
    * **Structural Similarity**: Measures the Jaccard similarity between editors' transition networks ($G_t^\alpha$) and the hyperlink structure.
    * **Editor Categorization**: Classifies users into **Specialists**, **Generalists**, and **Bots**.

## 🛠 Prerequisites
